# 08. Pilot Test Checklist (Field)

파일럿 현장에서 "폴더 선택 -> 스캔 -> 큐 -> 업로드 -> call 생성" 파이프의 안정성을 검증하기 위한 체크리스트.

## 사전 조건

- 지원 기기: Samsung One UI + Android 13/14
- 앱 로그인 완료(토큰 저장)
- SAF 통화녹음 폴더 선택 완료
- 테스트 파일: 소용량/대용량/중복 파일 세트 준비

## 테스트 항목 (15)

1. **기본 E2E 업로드 1건**
   - 기대 결과: `pending -> uploading -> uploaded -> synced`, `serverCallId` 생성
   - 실패 관찰: 어느 상태에서 멈췄는지, `errorMessage` 내용

2. **동일 파일 중복 스캔**
   - 기대 결과: 큐 중복 row 미생성(sha256 unique)
   - 실패 관찰: 동일 파일명/URI가 다중 row로 생기는지

3. **통화 직후 기록 중 파일**
   - 기대 결과: 안정화 전 파일은 skip, 다음 스캔에서만 큐 적재
   - 실패 관찰: 끊긴/부분 파일이 바로 업로드되는지

4. **연속 통화 파일 5건**
   - 기대 결과: 순차 큐 적재 및 업로드, 누락 없음
   - 실패 관찰: 중간 파일 누락/순서 꼬임

5. **네트워크 일시 끊김(업로드 중)**
   - 기대 결과: `retry_pending` 전환, retryCount 증가, 재연결 후 복구
   - 실패 관찰: 무한 uploading 고착 또는 재시도 없음

6. **완전 오프라인 상태 스캔/업로드**
   - 기대 결과: 스캔은 동작, 업로드는 실패 기록 후 재시도 대기
   - 실패 관찰: 앱 크래시 또는 큐 상태 손실

7. **토큰 만료(401)**
   - 기대 결과: refresh 1회 후 재요청, 성공 시 파이프 지속
   - 실패 관찰: refresh 성공에도 RELOGIN_REQUIRED 기록되는지

8. **refresh 실패**
   - 기대 결과: `RELOGIN_REQUIRED` 기록, 로그인 필요 상태로 전환
   - 실패 관찰: 무한 재시도 또는 원인 불명 에러

9. **앱 강제 종료 후 재실행**
   - 기대 결과: 선택 폴더/토큰/큐 상태 유지
   - 실패 관찰: SAF URI 유실, 대기 큐 초기화

10. **기기 재부팅 후 동작**
    - 기대 결과: 재실행 시 기존 큐 복원, 수동 워커 실행 가능
    - 실패 관찰: DB 손상/권한 유실

11. **절전 모드/배터리 최적화 ON**
    - 기대 결과: 지연은 가능하나 수동 트리거 시 업로드 가능
    - 실패 관찰: 워커 영구 미동작

12. **대용량 파일(예: 200MB+)**
    - 기대 결과: 스트리밍 업로드로 메모리 급증 없이 완료/실패 처리
    - 실패 관찰: OOM, 앱 프리즈, 업로드 중단

13. **서버 5xx 응답**
    - 기대 결과: retry_pending 전환 + errorMessage 기록
    - 실패 관찰: failed_upload로 즉시 전환되거나 상태 불일치

14. **최대 재시도 초과**
    - 기대 결과: `failed_upload` 최종 전환 + errorMessage 유지
    - 실패 관찰: retry_pending 무한 반복

15. **Status 운영 필터 검증**
    - 기대 결과: 실패/재시도 필요/업로드 중/완료 필터가 정확히 분류
    - 실패 관찰: 필터 결과와 DB 상태 불일치

## 운영 관찰 로그 포인트

- Settings의 `Step2(scan)` / `Step3(upload)` 로그
- Status 화면의 `status`, `retryCount`, `errorMessage`, `serverCallId`
- 서버의 `/api/mobile` 요청 로그(401/5xx/중복 처리 여부)
